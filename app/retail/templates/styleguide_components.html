Introduction

We primarily use bootstrap (v4.6) for our styling needs - please check the bootstrap (bs) documentation before adding any rules to our SCSS which could be handled with a bs utility class.

All of our SCSS is located under ./app/assets/v2/scss/**/*.scss, and we use django-compressor and libsass to compile our SCSS to CSS, in development, any change will trigger the document to be recompiled without needing any manual intervention.

If you are creating a new page, please ensure you include compress in the custom tag set and wrap your *.scss script(s) in a django-compressor block: 

[[djangoExample]]

Third-party libraries

In addition to bootstrap, we use the following third-party libraries:

  • Vue Select v3.10.8
  • Bootstrap Vue v2.18.0
  • Quill Editor v1.3.6
  • Font Awesome v5.12.1

Variables

In order to keep the system adaptable, we have ordered the inclusion of our SCSS documents such that ./gc-variables.scss can override the definitions of any other third-party library.

Our intention is to include all globally used libraries in ./lib/index.scss, which includes our variables before any third-party libraries. If you are introducing any new libraries to our system, please follow this procedure and add appropriate overides to ensure consistency.

If you wish to use these variables inside an .scss document, then please import ./gc-utilities.scss so that you have full access to everything mentioned in this document.

Variable Description
$blue / $gc-violet-400 * / $primary Purple
$green / $gc-teal-400 * / $secondary Teal
$pink / $gc-pink-400 * / $danger Pink
$yellow / $gc-yellow-400 * / $warning Yellow
$dark / $gc-purple Dark Purple
$light / $gc-grey Grey
$font-family-base 'Libre Franklin', sans-serif
$headings-font-family 'Miriam Libre', sans-serif
* Please note, we have each of these colors defined in different shades (100, 200, 300, 400 & 500), where 100 is the lightest and 500 is the darkest.

Utilities & Mixins

Along with the utilities/mixins available in bs we also offer a selection of additional utilities/mixins which can be found in ./gc-utilities.scss and ./gc-mixins.scss.

If you wish to extend these, please make sure we don't already have something similiar in bs or our codebase, if there is something similar as part of one of our custom pages, then please feel free to refactor and replace our custom rules with your generic utility.

Replacing old styles is an ongoing task and we would appreciate any and all help to get the codebase updated.

Class Options Description
.border-top-* [1-5] Places a solid border on the top of the element with a width of *px (1-5)
.border-left-* [1-5] Places a solid border on the left of the element with a width of *px (1-5)
.border-bottom-* [1-5] Places a solid border on the bottom of the element with a width of *px (1-5)
.border-right-* [1-5] Places a solid border on the right of the element with a width of *px (1-5)
.border-* [grey, primary, secondary, danger, info, dark, light] Sets the border-color property on an element
.top-0 / Sets top: 0px
.bottom-0 / Sets bottom: 0px
.hover-underline / Adds text-decoration: underline on ::hover
.text-primary-hover / Darkens .text-primary on ::hover
@include border() ($position, $minWidth, $maxWidth) Defines .border-*-* utility classes to set solid borders on elements
@include border-colors() ($colors...) Accepts key=>value pairs where key is the name and value is a color code to construct .border-*(color) utilities

Form Elements

If you are creating a new form, please use vue.js to build out the page, this will enable the full use of all components we have implemented.

Please note the use of .invalid in the following examples...

The .invalid class should be applied conditionally to the <form> based on the presence of errors after attempting to submit.

Wherever the ::invalid state is un-usable (because the input is nested inside the element we want to style) an additional .invalid class must be applied to the appropriate parent.
